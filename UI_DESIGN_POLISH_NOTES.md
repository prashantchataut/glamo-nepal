# GLAMO NEPAL UI Design Polish Notes

This pass moved the storefront away from an overbuilt, AI-generated feel and toward a calmer premium beauty-commerce direction.

## Design direction applied

- Reduced homepage density and removed several competing sections from the default homepage flow.
- Replaced the previous Dashain/dev-style banner with a customer-facing New Year 2083 offer banner.
- Simplified the hero into a more editorial beauty campaign layout.
- Simplified product cards by reducing badges, hiding noisy descriptions, and focusing on image, brand, name, rating, price, wishlist and add-to-cart.
- Simplified navigation and footer so the store feels more curated and less like a generated sitemap.
- Removed customer-facing implementation language such as frontend, config, mock, backend, placeholder and handoff from core customer routes where practical.
- Reworded checkout/cart/account/product copy so customers see shopping-oriented guidance instead of developer notes.
- Removed Framer Motion usage from global overlays/navigation to reduce bundle weight and make the UI feel calmer.

## Customer-facing banner direction

The primary campaign banner is now:

- New Year 2083 Special Offer
- Premium plum/pink/champagne palette
- Rich sale-card treatment inspired by the provided templates
- Customer CTA language: Shop New Year Offers, Explore New Arrivals

## Remaining visual dependencies

The UI will still look significantly better once GLAMO provides:

- real campaign photography or product photography
- supplier-approved product images
- brand-approved logos
- final campaign copy
- actual discount rules

Until then, the placeholder imagery is intentionally restrained and local to the project so the app avoids unsafe competitor hotlinks.
